# 해양/항만 관제 API 검토 메모

## 1. 현재 상태

- 공개 API 기준으로 `전국 실시간 개별 선박 위치 point`는 확인하지 못함
- 대신 `연안AIS 구역 통계`, `항로표지`, `조위/부이/해무 관측소`, `해상사격훈련구역`은 지도형으로 활용 가능
- 항만 시설은 `전국 공통`보다는 `울산항`, `인천항` 등 지역 항만공사 API가 더 현실적임

## 2. 바로 지도에 붙이기 좋은 후보

- [ ] `선박 밀집도` -> API 신청 대기 상태
  - API: [해양수산부_선박위치정보(연안AIS) 통계정보](https://www.data.go.kr/data/15084033/openapi.do)
  - 적합성: 높음
  - 이유: 1시간 단위 AIS 집계 정보를 `WMS/WFS`로 제공
  - 권장 사용: `선박 밀집도`, `해역 혼잡도`, `AIS 통계 히트맵`
  - 비고: 개별 선박 point가 아니라 `구역 단위 통계` 레이어

- [ ] `조위관측소`
  - API:
    - [해양수산부 국립해양조사원_조위관측소 최신 관측 데이터](https://www.data.go.kr/data/15038980/openapi.do)
    - [해양수산부 국립해양조사원_조위관측소 실측·예측 조위 조회](https://www.data.go.kr/data/15142507/openapi.do)
  - 적합성: 높음
  - 이유: 관측소 좌표와 조위 값을 제공
  - 권장 사용: `조위관측소`, `항만 조위 모니터링`
  - 비고: `GetSurveyTideLevelApiService`는 `obsCode`가 필요해 단독으로는 전국 레이어 구성이 어렵고, 관측소 목록/코드 API와 조인이 필요

- [ ] `해양관측부이` (신고 했음 cptkuk91@naver.com 메일 확인 필요)
  - API: [해양수산부 국립해양조사원_해양관측부이 최신 관측 데이터](https://www.data.go.kr/data/15038986/openapi.do)
  - 적합성: 높음
  - 이유: 부이 위치와 수온, 풍속, 파고 등 관측값 제공
  - 권장 사용: `실시간 해양관측`, `해상 환경 센서`
  - 비고: 항만 주변 해상 상태 모니터링에 적합

- [x] `해무관측소`
  - API:
    - [해양수산부 국립해양조사원_해무관측소 최신 관측데이터](https://www.data.go.kr/data/15099703/openapi.do)
    - [해양수산부 국립해양조사원_해무관측소 관측 데이터 조회](https://www.data.go.kr/data/15142519/openapi.do)
  - 적합성: 높음
  - 이유: 주요 무역항 인근 관측소 좌표와 시정, 풍속 등 제공
  - 권장 사용: `항만 시계/해무 위험`
  - 비고: 부산항, 인천항, 울산항 등 항만 주변 point 레이어에 적합

- [ ] `해상사격훈련구역`
  - API: [해양수산부 국립해양조사원_해상사격훈련구역](https://www.data.go.kr/data/15058193/openapi.do)
  - 적합성: 중상
  - 이유: 해상 위험구역을 polygon으로 표시 가능
  - 권장 사용: `해상 위험구역`, `사격훈련 통제구역`
  - 확인 포인트: 공식 설명상 좌표 정보 제공

- [ ] `항만시설`
  - API: [울산항만공사_위치기반 부두운영시설정보](https://www.data.go.kr/data/15156713/openapi.do)
  - 적합성: 중상
  - 이유: 위도/경도 기반 지역 항만시설 레이어 구성 가능
  - 권장 사용: `울산항 부두/선석 시설`
  - 비고: 전국 공통이 아니라 `지역 항만공사 단위`임

## 3. 조건부 후보

- [ ] `여객선 운항항로`
  - API:
    - [한국해양교통안전공단_운항항로 정보](https://www.data.go.kr/data/15142301/openapi.do)
    - [한국해양교통안전공단_운항 스케줄 정보](https://www.data.go.kr/data/15142302/openapi.do)
  - 적합성: 중간
  - 이유: 항로명, 스케줄, 운항 정보는 제공
  - 제한사항: API 단독으로는 지도 geometry가 약함
  - 구현 메모: 파일데이터인 [여객선 운항정보(KOMSA)](https://www.data.go.kr/data/15116479/fileData.do)와 결합하면 line/point 레이어 가능

- [ ] `항만 입출항/관제`
  - API:
    - [해양수산부_관제정보](https://www.data.go.kr/data/15006354/openapi.do)
    - [해양수산부_선박운항정보](https://www.data.go.kr/data/15006353/openapi.do)
  - 적합성: 중간
  - 이유: 입출항, 시설, 관제 작업 정보 자체는 유용
  - 제한사항: 공식 설명상 좌표가 핵심 필드로 보이지 않음
  - 구현 메모: 항구/시설 코드와 항만시설 좌표 데이터 조인이 필요할 가능성이 높음

## 4. 보조 또는 분석형 후보

- [ ] `실측 파랑`
  - API: [해양수산부 국립해양조사원_국가해양관측망 실측 파랑](https://www.data.go.kr/data/15155994/openapi.do)
  - 적합성: 보조
  - 이유: 파고, 파주기, 파향 등 분석값 제공
  - 권장 사용: `부이/관측소 상세 카드`, `파랑 경보 보조 지표`

- [ ] `격자별 해양정보`
  - API: [해양수산부 국립해양조사원_격자별 해양정보](https://www.data.go.kr/data/15002010/openapi.do)
  - 적합성: 보조
  - 이유: 해양 격자 단위 공간정보 제공
  - 권장 사용: 관측 데이터의 격자 기반 분석, 해역 정보 보조

## 5. 보류 또는 부적합

- [ ] `전국 실시간 선박 위치`
  - 상태: 공개 API 미확보
  - 비고: 전국 단위 개별 선박의 `실시간 point 위치`를 공개하는 공식 API는 확인하지 못함
  - 판단: 현재 가장 가까운 전국형 데이터는 [연안AIS 통계정보](https://www.data.go.kr/data/15084033/openapi.do)이며, 이는 `1시간 집계 구역 통계`임

- [ ] `실시간 개별 항적`
  - 상태: 공개 API 미확보
  - 비고:
    - [해양수산부_선박 AIS 동적정보](https://www.data.go.kr/data/15129186/fileData.do)
    - [해양수산부_(융합)선박 항적도](https://www.data.go.kr/data/15126803/fileData.do)
  - 판단: 과거 분석용 파일데이터에 가깝고 실시간 운영 레이어로는 부적합
