# 국가 인프라 API 검토 메모

기준일: 2026-03-06

## 1. 현재 mock 레이어

- `infra-grid-nodes`: 전력망 노드
- `infra-radiation-zone`: 원전 감시권

현재 구조상 국가 인프라 도메인은 `전력`, `원전`, `수자원`, `시설안전` 축으로 확장하는 것이 가장 현실적이다.

## 2. 결론

- 좋은 API는 존재한다.
- 다만 `전력망 노드`를 그대로 대체할 공개 좌표형 API는 약하다.
- 반대로 `원전 방사선`, `원전 상태`, `공공시설물 안전`, `댐/수문 운영`은 바로 구현 가능한 공식 API가 있다.

## 4. 구현 판단

### 4.1 전력망 노드

- 공개 API만으로 `변전소/송전망 좌표`를 바로 받는 경로는 현재 확인되지 않았다.
- 가장 가까운 후보는 `한국전력공사 분산전원 연계정보`다.
- 다만 `변전소명`, `변전소코드`, `용량` 중심이라 지도 마커로 바로 쓰기 어렵다.
- 따라서 `전력망 노드`는 당장 실데이터로 1:1 대체하기보다 아래 두 방향 중 하나가 현실적이다.

1. `현재전력수급현황조회`를 사용해 전력 메트릭/알림으로 전환
2. `공공시설물 안전관리 정보조회`로 국가 인프라 지도 레이어 자체를 먼저 실데이터화

### 4.2 원전 감시권

- `원자력발전소 실시간 주변 방사선량` API는 실측값을 제공하므로 실데이터 반영이 가능하다.
- 다만 polygon 형태의 `감시권` 자체를 내려주지는 않는다.
- 따라서 권장 구조는 아래와 같다.

1. 원전 본부 위치/권역 polygon은 정적 관리
2. API에서는 실시간 방사선량과 원전 상태만 수집
3. polygon 클릭 시 최신 방사선량/호기 상태를 패널에 표기

## 5. 권장 구현 순서

1. `공공시설물 안전관리 정보조회 서비스`
- 좌표가 있어 바로 지도 레이어 구현 가능
- 국가 인프라 도메인 실데이터화에 가장 빠름

2. `원자력발전소 실시간 주변 방사선량` + `원자력 발전소 상태`
- 현재 `원전 감시권` mock을 가장 자연스럽게 고도화 가능

3. `수문 운영 정보` 또는 `다목적댐 운영 정보`
- 전력/원전 외 인프라 축 추가

4. `현재전력수급현황조회`
- 지도 레이어보다 메트릭/경보 카드용으로 우선 적용

## 6. 최종 판단

- 바로 구현하기 가장 좋은 API:
  - `국토안전관리원_공공시설물 안전관리 정보조회 서비스`
  - `한국수력원자력(주)_원자력발전소 실시간 주변 방사선량`
  - `한국수력원자력(주)_원자력 발전소 상태`

- 보조 지표용으로 좋은 API:
  - `한국전력거래소_현재전력수급현황조회`
  - `한국수자원공사_수문 운영 정보`

- 아직 한계가 큰 API:
  - `한국전력공사 분산전원 연계정보`
  - 이유: 좌표형 공개 데이터가 약해 `전력망 노드` 대체가 불완전함
