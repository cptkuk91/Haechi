# 보건/의료 API 검토 메모

기준일: 2026-03-07

검토 기준:
- Perplexity + Brave Search로 확인
- 공식 문서/공공데이터포털 기준으로만 정리
- Data Layer에 바로 붙일 수 있는 `위도/경도`, `주소`, `공간정보` 보유 여부를 우선 확인

## 1. 현재 상태

- [x] `응급실 위치`
  - API: [국립중앙의료원_전국 응급의료기관 정보 조회 서비스](https://www.data.go.kr/data/15000563/openapi.do)
  - 상태: upstream 정상 표시
  - 비고: 현재 `/api/health/emergency-rooms` 응답이 `source=upstream` 상태로 확인됨

- [x] `외상센터`
  - API: [국립중앙의료원_전국 응급의료기관 정보 조회 서비스](https://www.data.go.kr/data/15000563/openapi.do)
  - 상태: upstream 정상 표시
  - 비고: 현재 `/api/health/trauma-centers` 응답이 `source=upstream` 상태로 확인됨

- [x] `응급실 수용 현황`
  - API: [국립중앙의료원_전국 응급의료기관 정보 조회 서비스](https://www.data.go.kr/data/15000563/openapi.do)
  - 상태: upstream 정상 표시
  - 비고: `getEmrrmRltmUsefulSckbdInfoInqire` 실시간 가용병상 데이터를 `hpid` 기준으로 응급실/외상센터 위치에 조인했고, 병원 상세 패널은 `/api/health/facility-status`에서 `수용률/가용병상`을 계산해 표시함

- [ ] `구급차 이동 경로`
  - 상태: mock 유지 중
  - 비고: 공개 실시간 차량 위치 API는 아직 미확보

- [ ] `감염 위험 구역`
  - 상태: mock 유지 중
  - 비고: 공개 전국형 실시간 polygon API는 별도 검토 필요

## 2. 전국형 지도 레이어 후보

- [ ] `자동심장충격기(AED)`
  - API: [국립중앙의료원_전국 자동심장충격기(AED) 정보 조회 서비스](https://www.data.go.kr/data/15000652/openapi.do)
  - 적합성: 높음
  - 이유: 전국 단위 응급 장비 위치 레이어로 바로 확장 가능
  - 확인 포인트: 주소/설치장소/기관명 기반 point 레이어 구성 가능

- [ ] `약국 위치`
  - API: [국립중앙의료원_전국 약국 정보 조회 서비스](https://www.data.go.kr/data/15000576/openapi.do)
  - 적합성: 높음
  - 이유: 보건/의료 도메인에서 가장 자연스러운 생활 접근성 레이어
  - 확인 포인트: 주소 기반 조회, 진료요일/시간대 조건 지원

- [ ] `병·의원 위치`
  - API: [국립중앙의료원_전국 병·의원 찾기 서비스](https://www.data.go.kr/data/15000736/openapi.do)
  - 적합성: 높음
  - 이유: 전국 의료기관 레이어 기본축으로 사용 가능
  - 확인 포인트: 주소, 진료과목, 기관구분 기반 필터링 가능

- [ ] `생활안전지도 약국`
  - API: [행정안전부_생활안전지도 약국](https://www.data.go.kr/data/15149539/openapi.do)
  - 적합성: 중간
  - 이유: 공식 문서상 지도 표시 가능한 공간정보와 주소/전화/진료시간 제공
  - 비고: NMC 약국 API와 중복 가능성이 있어 보조 소스로 보는 편이 맞음

- [ ] `응급의료시설 WMS`
  - API: [행정안전부_응급의료시설](https://www.data.go.kr/data/15101855/openapi.do)
  - 적합성: 중간
  - 이유: 응급의료시설 위치/상세정보 제공
  - 비고: WMS 기반이라 기존 point 레이어와는 구현 방식이 다름

## 3. 값 전용 또는 보조 API

- [ ] `의료기관 실시간 병상정보`
  - API: [행정안전부_의료기관 실시간 병상정보](https://www.data.go.kr/data/15153443/openapi.do)
  - 적합성: 보조
  - 이유: 실시간 병상 값은 유용하지만 좌표형 메인 레이어로는 부족
  - 권장 사용: `응급실 위치` 또는 `병원 레이어`에 수용 정보 조인

- [x] `응급실 실시간 가용병상`
  - API: [국립중앙의료원_전국 응급의료기관 정보 조회 서비스](https://www.data.go.kr/data/15000563/openapi.do)
  - 적합성: 보조
  - 이유: `getEmrrmRltmUsefulSckbdInfoInqire`가 병상/중환자실/장비 가용량을 제공
  - 권장 사용: `응급실 위치` 및 `외상센터` 레이어와 `hpid`로 조인

- [ ] `병원 상세정보 보강`
  - API: [건강보험심사평가원_병원정보서비스](https://www.data.go.kr/data/15001698/openapi.do)
  - API: [건강보험심사평가원_의료기관별상세정보서비스](https://www.data.go.kr/data/15001699/openapi.do)

  - 권장 사용: 병원 기본 목록 확보 후 상세 패널 정보 보강

## 8. 주의사항

- 주소 기반 API를 좌표로 바꾸기 위해 [국토교통부_지오코더 API](https://www.data.go.kr/data/15101106/openapi.do)를 사용할 수는 있다.
- 다만 공식 문서상 지오코딩 결과는 `실시간 사용` 전제이며 `별도의 저장장치나 데이터베이스에 저장할 수 없음`이 명시되어 있어, DB 캐시형 파이프라인의 기본 좌표 소스로 삼는 것은 부적절하다.
- 따라서 보건/의료 레이어는 가능한 한 원본 API 자체에 `위도/경도` 또는 `공간정보`가 있는 후보부터 붙이는 것이 맞다.
