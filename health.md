# 보건/의료 API 검토 메모

기준일: 2026-03-08

검토 기준:
- Perplexity + Brave Search로 확인
- 공식 문서/공공데이터포털 기준으로만 정리
- Data Layer에 바로 붙일 수 있는 `위도/경도`, `주소`, `공간정보` 보유 여부를 우선 확인

## 1. 현재 상태

- [x] `시도별 감염 위험도`
  - 상태: upstream 연동 및 시도별 choropleth 구현 완료
  - API: [질병관리청_전수신고 감염병 발생현황](https://www.data.go.kr/data/15139178/openapi.do) `/Region`
  - 비고: 지역별(시도) 감염병 발생 현황(발생수, 인구 10만명당 발생률)을 사용해 `시도별 choropleth` 레이어 구현 완료. 연도/질병/지표 selector 지원
  - 조인용 공간정보: [국토교통부_시군구](https://www.data.go.kr/data/15059464/openapi.do)
  - 조인용 공간정보 비고: 행정구역도(광역시도/시군구/읍면동/리) 공간정보를 제공하므로 시도 경계와 조인 가능. 현재 구현은 광역시도 polygon 기준
  - 제한사항: `공식 위험 구역 polygon`을 바로 내려주는 API는 아니며, 실시간이 아니라 시도 단위 통계 조인 방식임

- [x] `기간별 감염 추세`
  - 상태: upstream 연동 및 분석 패널 구현 완료
  - API: [질병관리청_전수신고 감염병 발생현황](https://www.data.go.kr/data/15139178/openapi.do) `/PeriodBasic`, `/PeriodRegion`, `/PeriodPtnt`
  - 비고: 기간별(연도, 월) 발생 현황을 중심으로, 국내/국외 감염지역별 현황, 환자/의사환자/병원체보유자 분류별 현황을 사용해 `전체 추세`, `감염지역별`, `환자분류별` 분석 패널 구현 완료
  - 권장 사용: 지도 polygon보다는 보건/의료 분석 패널, 대시보드 차트, 트렌드 요약 카드에 적합

- [x] `감염병 상세 분포`
  - 상태: upstream 연동 및 분석 패널 구현 완료
  - API: [질병관리청_전수신고 감염병 발생현황](https://www.data.go.kr/data/15139178/openapi.do) `/Disease`, `/Age`, `/Gender`, `/death`
  - 비고: 감염병별 발생 수/발생률, 연령별, 성별, 사망 현황을 묶어 `감염병별`, `연령별`, `성별`, `사망 현황` 분석 패널 구현 완료. 연도/지표/감염병 selector 지원
  - 권장 사용: 지도 레이어 자체보다는 필터, 툴팁, 우측 상세 패널, 요약 통계 영역에 적합

## 2. 전국형 지도 레이어 후보

- [ ] `자동심장충격기(AED)`
  - API: [국립중앙의료원_전국 자동심장충격기(AED) 정보 조회 서비스](https://www.data.go.kr/data/15000652/openapi.do)
  - 적합성: 높음
  - 이유: 전국 단위 응급 장비 위치 레이어로 바로 확장 가능
  - 확인 포인트: 주소/설치장소/기관명 기반 point 레이어 구성 가능

- [ ] `약국 위치`
  - API: [국립중앙의료원_전국 약국 정보 조회 서비스](https://www.data.go.kr/data/15000576/openapi.do)
  - 적합성: 높음
  - 이유: 보건/의료 도메인에서 가장 자연스러운 생활 접근성 레이어
  - 확인 포인트: 주소 기반 조회, 진료요일/시간대 조건 지원

- [ ] `병·의원 위치`
  - API: [국립중앙의료원_전국 병·의원 찾기 서비스](https://www.data.go.kr/data/15000736/openapi.do)
  - 적합성: 높음
  - 이유: 전국 의료기관 레이어 기본축으로 사용 가능
  - 확인 포인트: 주소, 진료과목, 기관구분 기반 필터링 가능

- [ ] `생활안전지도 약국`
  - API: [행정안전부_생활안전지도 약국](https://www.data.go.kr/data/15149539/openapi.do)
  - 적합성: 중간
  - 이유: 공식 문서상 지도 표시 가능한 공간정보와 주소/전화/진료시간 제공
  - 비고: NMC 약국 API와 중복 가능성이 있어 보조 소스로 보는 편이 맞음

- [ ] `응급의료시설 WMS`
  - API: [행정안전부_응급의료시설](https://www.data.go.kr/data/15101855/openapi.do)
  - 적합성: 중간
  - 이유: 응급의료시설 위치/상세정보 제공
  - 비고: WMS 기반이라 기존 point 레이어와는 구현 방식이 다름

## 3. 값 전용 또는 보조 API

- [ ] `의료기관 실시간 병상정보`
  - API: [행정안전부_의료기관 실시간 병상정보](https://www.data.go.kr/data/15153443/openapi.do)
  - 적합성: 보조
  - 이유: 실시간 병상 값은 유용하지만 좌표형 메인 레이어로는 부족
  - 권장 사용: `응급실 위치` 또는 `병원 레이어`에 수용 정보 조인

- [ ] `병원 상세정보 보강`
  - API: [건강보험심사평가원_병원정보서비스](https://www.data.go.kr/data/15001698/openapi.do)
  - API: [건강보험심사평가원_의료기관별상세정보서비스](https://www.data.go.kr/data/15001699/openapi.do)
  - 적합성: 보조
  - 이유: 상세 속성 보강에는 좋지만 지도형 좌표 소스 여부는 추가 확인 필요
  - 권장 사용: 병원 기본 목록 확보 후 상세 패널 정보 보강

## 4. 지역형 확장 후보

- [ ] `보건소 현황`
  - API: [경기도_보건소 현황](https://www.data.go.kr/data/15057633/openapi.do)
  - 적합성: 지역형
  - 이유: 공식 설명상 도로명주소, 위도, 경도 포함

- [ ] `공중보건의 배치 현황`
  - API: [경기도_보건 의료기관별 공중보건의 배치 현황](https://www.data.go.kr/data/15056592/openapi.do)
  - 적합성: 지역형
  - 이유: 보건소/보건지소 + 인력 배치 + 위도/경도 포함

- [ ] `어린이 예방접종 지정의료기관`
  - API: [경기도_어린이 국가 예방접종 지정의료기관 현황](https://www.data.go.kr/data/15057195/openapi.do)
  - 적합성: 지역형
  - 이유: 지정의료기관 위치 레이어로 확장 가능

## 5. API로 바로 보기 어려운 보완 후보

- [ ] `건강증진센터 / 정신건강증진센터 / 자살예방센터`
  - 데이터: [전국건강증진센터표준데이터](https://www.data.go.kr/data/15021137/standard.do)
  - 적합성: 높음
  - 이유: 공식 표준데이터에 도로명주소, 지번주소, 위도, 경도 포함
  - 비고: OpenAPI보다는 표준데이터/파일 기반 적재 쪽이 현실적

## 8. 주의사항

- 주소 기반 API를 좌표로 바꾸기 위해 [국토교통부_지오코더 API](https://www.data.go.kr/data/15101106/openapi.do)를 사용할 수는 있다.
- 다만 공식 문서상 지오코딩 결과는 `실시간 사용` 전제이며 `별도의 저장장치나 데이터베이스에 저장할 수 없음`이 명시되어 있어, DB 캐시형 파이프라인의 기본 좌표 소스로 삼는 것은 부적절하다.
- 따라서 보건/의료 레이어는 가능한 한 원본 API 자체에 `위도/경도` 또는 `공간정보`가 있는 후보부터 붙이는 것이 맞다.
